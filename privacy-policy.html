Semsa Privacy Policy

Last updated: 2026-04-17

1) Overview

Semsa is designed for local-first AI usage on your device. Core chat, memory, and voice features are intended to run on-device whenever possible.

2) Data We Process

Depending on features you use, Semsa can process:

  • Text you type in chat and translator modes
  • Voice audio captured from microphone while voice input is active
  • Files you explicitly upload (documents, images, audio)
  • Local memory entries you ask Semsa to store (for example reminders, birthdays, notes)
  • Technical diagnostics needed to keep app features working

3) On-Device Processing

Semsa processes AI tasks locally on your device by default (LLM, STT/TTS runtime, local memory handling).

4) Network Access

Semsa uses internet only when needed, for example:

  • Downloading required runtime models/assets
  • Refreshing model/asset metadata

Some third-party SDKs may contact their servers for updates, compatibility data, and operational metrics.

5) Third-Party Components

Semsa integrates third-party AI/runtime libraries and models. Their licenses and terms can apply in addition to this policy.

6) Permissions

Semsa currently requests:

  • Microphone (RECORD_AUDIO): for voice input features
  • Notifications (POST_NOTIFICATIONS): for reminder notifications
  • Internet (INTERNET): for model/asset download and related updates

7) Data Sharing

Semsa does not sell personal data. If a feature requires off-device transfer through a third-party SDK/service, this must be disclosed in product documentation and Play Data Safety declarations.

8) Storage and Retention

Local chat, memory, and downloaded model files are stored on-device. You can remove chats/files/memory entries from the app UI where available.

9) Your Controls

You can:

  • Avoid voice features by not granting microphone permission
  • Delete chat sessions and stored files in sidebars/tools
  • Remove local models/assets when delete options are provided

10) Security

We apply practical safeguards for local storage and runtime handling, but no software can guarantee absolute security.

11) Children

If Semsa is ever distributed to children, additional policy and parental safeguards may apply.

12) Changes

We may update this policy as features or legal requirements evolve. Significant changes should update the "Last updated" date.

13) Contact
