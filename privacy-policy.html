Shemsa Privacy Policy

Last updated: 2026-07-02

1) Overview

Shemsa is designed for local-first AI usage on your device. Core chat, memory, translation, smart home, and voice features are intended to run on-device whenever possible.

2) Data We Process

Depending on the features you use, Shemsa can process:

  • Text you type in chat, translator, memory, learning, coding, and smart home features
  • Voice audio captured from the microphone while voice input or wake-word features are active
  • Files you explicitly upload or capture, such as documents, images, and audio
  • Local memory entries you ask Shemsa to store, such as notes, reminders, birthdays, anniversaries, and personal facts
  • Approximate or precise location only when you allow location-based weather features
  • Smart home device information you add locally, such as device address, device keyword/parser phrases, and local device status
  • Technical diagnostics needed to keep app features working

3) On-Device Processing

Shemsa processes AI tasks locally on your device by default, including the local LLM, speech-to-text/text-to-speech runtime where available, local memory handling, and local smart home commands where supported.

4) Network Access

Shemsa uses internet only when needed, for example:

  • Downloading required runtime models/assets
  • Refreshing model/asset metadata
  • Fetching weather data when weather features are used
  • Fetching news, Wikipedia/knowledge, or website content when internet-based tools are used
  • Processing optional Google Play Billing support purchases

5) Third-Party Components

Shemsa integrates third-party AI/runtime libraries, models, and public information services. Their licenses, terms, and availability can apply in addition to this policy.

Examples may include model hosting/download providers, weather providers, news or knowledge sources, Google Play Billing, and Android system services.

6) Permissions

Shemsa currently requests permissions as needed:

  • Microphone (RECORD_AUDIO): for voice input, translation voice features, and optional wake-word/foreground voice commands
  • Camera (CAMERA): for camera capture, image input, OCR/image-related features, and optional smart camera workflows
  • Location (ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION): for weather based on your current location and Wi-Fi/smart home setup flows when Android requires location-related access for network discovery
  • Notifications (POST_NOTIFICATIONS): for reminder notifications and foreground voice command status
  • Internet and network state: for model/asset downloads, weather, news/knowledge tools, billing, and related updates
  • Wi-Fi/network permissions: for local smart home discovery and setup
  • Exact alarms/full-screen intent/vibration: for reminders and alarm-style reminder notifications where supported

7) Data Sharing

Shemsa does not sell personal data.

Most user content is stored and processed locally on your device. Some optional features require network requests:

  • Weather may send location coordinates or selected location information to a weather provider.
  • News, Wikipedia/knowledge, and website tools request public content from internet services.
  • Model downloads request files from model/asset hosting providers.
  • Google Play Billing processes optional support purchases through Google Play.
  • Smart home control is designed for local network control where supported and does not require Shemsa cloud control.

8) Storage and Retention

Local chats, memory entries, uploaded/captured files, smart home settings, and downloaded model files are stored on-device. You can remove chats/files/memory entries from the app UI where available, clear app storage in Android settings, or uninstall the app.

9) Your Controls

  • Avoid voice features by not granting microphone permission
  • Avoid camera features by not granting camera permission
  • Avoid location-based weather by not granting location permission
  • Delete chat sessions and stored files in sidebars/tools
  • Delete local memory entries where available
  • Remove local app data by clearing app storage or uninstalling the app
  • Avoid optional internet tools such as weather, news, Wikipedia/knowledge, and smart home setup

10) Data Deletion Requests

Shemsa does not require users to create an account. Most Shemsa data, including chats, memories, uploaded files, smart home settings, and downloaded model assets, is stored locally on your device.

You can delete local app data by using the deletion controls provided in the app, clearing the app's storage in Android settings, or uninstalling the app.

If you need help deleting app-related data or want to request deletion of data associated with Shemsa, contact us at semsa.app@gmail.com. Please include enough information for us to understand your request. We will respond to deletion requests as required by applicable law.

11) Security

We apply practical safeguards for local storage and runtime handling, but no software can guarantee absolute security.

12) Children

If Shemsa is ever distributed to children, additional policy and parental safeguards may apply.

13) Changes

We may update this policy as features or legal requirements evolve. Significant changes should update the "Last updated" date.
